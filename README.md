# Customer Payment & Location Analysis (SQL)

## Dataset Overview
This project uses three relational tables:
- customer (599 rows)
- payment (14,596 rows)
- address (603 rows)

## Objective
To analyze customer payment behavior and rank customers by total revenue within each district using SQL joins and aggregations.

## Tables Used
- customer
- payment
- address

## Key Concepts Used
- INNER JOIN
- GROUP BY
- Aggregate functions (SUM)
- ORDER BY for ranking
- Business-driven SQL analysis
